# Spec: Documentation Foundation

## Problem Statement
The `docs/foundation/` directory needs all nine reference documents before implementation sprints begin. These documents serve as the project's durable memory and the professor's evidence of process.

## Acceptance Criteria
1. `student-reading-guide.md` exists and explains where to start
2. `orchestration-method.md` exists and documents the six-step workflow
3. `site-design-philosophy.md` exists and documents Bauhaus, Sage, and Authority decisions
4. `site-implementation-codex.md` exists and defines all coding conventions
5. `site-spec-patterns.md` exists with spec and sprint templates
6. `power-words.md` exists with all named framework terms defined
7. `technology-orientation.md` exists with rationale for each technology
8. `verification-and-deployment.md` exists with all quality commands
9. `lightweight-change-path.md` exists with the Change Note template

## Scope
### In Scope
- All 9 files in `docs/foundation/`

### Out of Scope
- Spec artifacts in `docs/_specs/`
- Research materials in `docs/_research/`
- Any application code

## Dependencies
- None — documentation can be written before code
